Прекратите использовать JavaScript!

На данной странице слово "загрузить" используется в значении "получить по протоколу IP файл с удалённого компьютера"

Веб-страница представляет собой HTML-документ, в котором прописан текст, который видит пользователь, описано то, как страница будет выглядеть, указываются ссылки на все элементы, которые браузеру необходимо загрузить с сервера вслед за самой страницей (HTML-документом). В 1995 году появился JavaScript (точнее, в то время он имел иное название), который позволял менять страницу прямо на компьютере пользователя. По сути, JS-код это программа, взаимодействующая с API браузера, а через него со страницей, которую вы видите. Можно добавить, удалить, изменить любой элемент. Однако не стоит забывать, что это происходит у вас на компьютере, а идеологически все операции по преобразованию должны происходить на сервере: to serve в переводе с английского - обслуживать, server - тот, кто обслуживает, в нашем случае высокопроизводительный компрьютер (мейнфрейм). Он способен найти в базе данных и отобразить (тех. более корректно: отправить вам) информацию о вашей учётной записи на сайте, который обслуживается сервером, принять во внимание ваши настройки и уровни доступа, и, скажем, сайт электронной почты выведет список только ваших писем и выделить их пурпурным цветом (естественно, если вы укажете это в настройках). Обратите внимание: не важно на сколько у вас производительный компьютер (или это вовсе не компьютер), страница обработается мощным сервером и останется только вывести её на экран. На любом устройстве страница загрузится одинаково быстро при условии одинаковой скорости соединения.

Однако, как это делается в наше время, подобного рода задачи выполняются с помощью JavaScript на компьютере пользователя. Это значит что на более мощном клиенте (устройство пользователя) обработка завершится раньше, на более слабом - позже. Идёт время, в язык Javascript добавляются новые возможности, новые стандарты языка реализуются в новых версиях браузеров, которые в свою очередь, если сравнить с предыдущими, разрастаются в размере и становятся более требовательными. Таким образом, современная JS-страница не отобразится корректно в браузере 3-летней давности.
В итоге придётся регулярно менять старый ПК на новый ради просмотра JS-страниц. Здорово, да?

Конечно, есть исключительные ситуации, когда действительно есть нехобходимость динамического изменения страницы, но это не загрузка картинки с cdn-домена. Кстати, в браузере есть встроенный медиаплеер, нет нужды писать свой на джаваскрипте (который нужно ещё отдельно загружать с третьей стороны). Пожалуйста, не изобретайте велосипед, прекратите использовать JavaScript где попало.
